# 🤝 Contributing to ProjectFlow

First, thank you for taking the time to contribute! We love community input.



## 🛠 Local Development
To set up a development environment:
1.  **Fork** the repo and **Clone** your fork.
2.  Follow the setup instructions in the `README.md`.
3.  Ensure your `.env` file is properly configured.

## 🌿 Branching Policy
We use the **GitHub Flow** model:
* Always create a new branch for your work: `git checkout -b feat/your-feature-name`.
* Avoid committing directly to `main`.

## 📝 Commit Messages
Keep them descriptive and prefix them with the type of change:
* `feat:` (new feature)
* `fix:` (bug fix)
* `docs:` (documentation changes)
* `style:` (formatting, missing semi-colons, etc)

## 🐛 Reporting Issues
If you find a bug, please open an **Issue** and include:
* A clear description of the bug.
* Steps to reproduce.
* Relevant logs from `docker-compose logs`.

## 🚀 Submitting a Pull Request
1.  Push your branch to your fork.
2.  Open a Pull Request against our `main` branch.
3.  Describe your changes in detail and link any related issues.
